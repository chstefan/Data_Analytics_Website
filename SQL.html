Google Play Store Data Analysis and Database Management

Project Overview

In this project, I embarked on a comprehensive data cleaning, analysis, and database creation process using a dataset obtained from the Google Play Store. The objective was to refine the dataset for better analysis, implement it into a relational database, and perform queries to extract meaningful insights regarding app trends and performance.

Data Preprocessing with Python

The first phase involved using Python's pandas library and a custom size conversion function to clean the dataset. The steps included reading the data, casting columns to the correct data types, parsing dates, and dropping duplicates. The cleaned data was saved as 'googleplaystore_cleaned.csv'.

Database Creation and Data Insertion using PostgreSQL

A PostgreSQL database called google_play_store was created, and a table apps was structured to contain the dataset attributes. Data from the cleaned CSV was imported into the database using the \copy command.

Data Analysis Queries

Several SQL queries were executed to derive insights such as category distribution, average ratings, top installed apps, content rating distribution, and top apps by category and type, using advanced SQL features like window functions.

Conclusion

This project demonstrates my comprehensive data analytics capabilities, combining Python and SQL to prepare and analyze data. The detailed process and findings are a testament to my professional approach to data analysis and management.

SQL Queries Executed

Query Description SQL Command
Category Distribution SELECT category, COUNT(*) AS number_of_apps FROM apps GROUP BY category ORDER BY number_of_apps DESC;
Average Ratings by Category SELECT category, AVG(rating) AS avg_rating FROM apps GROUP BY category ORDER BY avg_rating DESC;
Top Installed Apps SELECT app_name, installs FROM apps ORDER BY installs DESC LIMIT 10;
Content Rating Distribution SELECT content_rating, COUNT(*) AS number_of_apps FROM apps GROUP BY content_rating ORDER BY number_of_apps DESC;
Top Apps by Category and Type WITH ranked_apps AS (SELECT category, type, app_name, installs, ROW_NUMBER() OVER(PARTITION BY category, type ORDER BY installs DESC) AS rank FROM apps) SELECT category, type, app_name, installs FROM ranked_apps WHERE rank = 1 ORDER BY category, type;
